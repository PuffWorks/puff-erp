# Puff ERP 使用手册与 QA 帮助中心

## 一、系统简介

Puff ERP 是一套面向服务器、硬件设备、IT 代理商、系统集成商的业务管理系统。

系统覆盖：

* CRM 客户管理
* 数据公海
* 我的客户
* 联系人管理
* 跟进记录
* 商机管理
* 报价管理
* 销售订单
* 缺货采购
* 采购订单
* 入库管理
* 出库管理
* 库存现存量
* 库存流水
* SN 序列号管理
* 应收、应付、收款、付款
* 合同管理
* 售后工单
* 质保查询
* 报表看板
* 消息待办
* 导入导出
* 附件管理
* 打印
* 权限管理
* 系统授权

系统的核心设计是三条线：

```text
货：库存、SN、入库、出库、盘点、库存流水
钱：应收、应付、收款、付款、核销
单：报价、销售、采购、合同、售后、审批、日志
```

所有业务尽量通过单据驱动，避免直接修改库存、金额和状态，确保数据可追溯。

# 二、用户角色说明

## 1. 超级管理员

主要负责：

* 用户管理
* 角色管理
* 菜单管理
* 权限分配
* 系统配置
* 系统授权
* 数据字典
* 编号规则
* 登录日志
* 操作日志
* 文件管理
* 数据权限配置

适合对象：

```text
系统负责人、技术管理员、实施人员
```

---

## 2. 销售人员

主要负责：

* 领取公海客户
* 维护我的客户
* 新增联系人
* 记录客户跟进
* 创建商机
* 创建报价单
* 报价转销售订单

销售一般不能查看：

* 成本价
* 采购价
* 毛利
* 库存金额
* 其他销售的客户数据

---

## 3. 销售主管

主要负责：

* 查看团队客户
* 查看团队商机
* 审核报价单
* 审核销售订单
* 分配公海客户
* 查看团队销售数据
* 查看销售漏斗和 CRM 看板

---

## 4. 采购人员

主要负责：

* 处理缺货采购
* 创建采购订单
* 采购入库跟进
* 查看供应商
* 查看采购订单状态
* 维护采购业务附件

---

## 5. 仓库人员

主要负责：

* 查看库存现存量
* 确认采购入库
* 确认销售出库
* 处理 SN 入库和出库
* 库存盘点
* 查看库存流水
* 处理售后备件出库，如已启用

仓库人员一般不能查看：

* 销售毛利
* 客户回款
* 采购成本以外的财务数据
* 非库存相关敏感信息

---

## 6. 财务人员

主要负责：

* 应收管理

## 7. 售后人员

## 8. 老板 / 管理层

# 三、系统登录与基础操作

## 1. 如何登录系统？

操作步骤：

1. 打开系统地址。
2. 输入账号。
3. 输入密码。
4. 输入验证码。
5. 点击“登录”。

如果登录失败，请检查：

* 账号是否正确
* 密码是否正确
* 验证码是否过期
* 用户是否被停用
* 系统授权是否有效

---

## 2. 忘记密码怎么办？

请联系系统管理员重置密码。

管理员可在：

```text
系统管理 → 用户管理 → 重置密码
```

中进行操作。

---

## 3. 如何修改自己的密码？

进入：

```text
个人中心 → 修改密码
```

输入旧密码和新密码后保存。

---

## 4. 为什么我看不到某个菜单？

常见原因：

1. 当前角色没有该菜单权限。
2. 当前系统授权未包含该模块。
3. 管理员未分配按钮权限。
4. 菜单被停用。
5. 用户所属角色配置错误。

请联系管理员检查：

```text
用户角色
角色菜单
模块授权
按钮权限
```

---

# 四、CRM 中心使用说明

## 1. CRM 中心包含哪些功能？

CRM 中心包含：

```text
CRM看板
数据公海
我的客户
联系人管理
跟进记录
商机管理
销售任务
线索管理
```

---

## 2. 数据公海是什么？

数据公海是未分配给具体销售的客户池。

适合存放：

* 新导入的客户
* 未明确负责人的客户
* 被销售退回的客户
* 主管待分配的客户
* 市场线索转化后的待分配客户

销售可以根据权限领取公海客户。

主管或管理员可以批量分配客户。

---

## 3. 我的客户是什么？

我的客户是已经分配到当前销售或当前团队名下的客户。

销售可以对自己的客户进行：

* 查看
* 编辑
* 新增联系人
* 添加跟进记录
* 创建商机
* 创建报价
* 打标签
* 退回公海
* 转移客户，按权限控制

---

## 4. 数据公海支持哪些批量操作？

数据公海支持：

```text
批量领取
批量分配
批量打标签
批量删除
```

说明：

* 批量领取：将公海客户领取到当前销售名下。
* 批量分配：主管或管理员将公海客户分配给指定销售。
* 批量打标签：给选中的客户统一添加标签。
* 批量删除：删除无业务关联的公海客户，通常仅管理员可操作。

注意：

```text
已有报价、订单、合同、售后、财务记录的客户不建议删除。
```

---

## 5. 我的客户支持哪些批量操作？

我的客户支持：

```text
批量转移
退回公海
批量打标签
```

说明：

* 批量转移：把客户转给其他销售。
* 退回公海：将客户从私有客户退回到公海池。
* 批量打标签：给多个客户统一添加标签。

---

## 6. 客户标签有什么用？

客户标签用于快速识别客户特征。

例如：

```text
重点客户
服务器需求
网络设备需求
老客户
高意向
待回访
价格敏感
售后风险
```

客户标签可以用于：

* 客户筛选
* 销售跟进
* 批量运营
* CRM 看板分析
* 客户分层管理

---

## 7. 什么是联系人？

联系人是客户公司中的具体人员。

一个客户可以有多个联系人，例如：

```text
采购负责人
技术负责人
财务负责人
老板
项目负责人
售后联系人
```

联系人可用于报价、订单、合同、售后等业务。

---

## 8. 什么是跟进记录？

跟进记录用于记录销售与客户之间的沟通过程。

常见跟进方式：

```text
电话
微信
邮件
拜访
会议
远程沟通
其他
```

建议每次跟进都填写：

* 跟进内容
* 客户反馈
* 下一步动作
* 下次跟进时间

这样可以避免客户长期无人跟进，也方便销售交接。

---

## 9. 什么是商机？

商机表示一个可能成交的业务机会。

例如：

```text
某客户计划采购 5 台服务器
某客户准备做网络升级
某客户需要存储扩容
```

商机通常包含：

* 客户
* 联系人
* 预计金额
* 预计成交日期
* 当前阶段
* 成功率
* 需求描述
* 竞争对手
* 输单原因

商机成熟后，可以转为报价单。

---

## 10. 商机阶段如何理解？

常见阶段：

```text
初步接触
需求确认
方案阶段
报价阶段
商务谈判
赢单
输单
暂停
```

商机阶段用于销售漏斗分析。

---

# 五、报价管理使用说明

## 1. 报价单的作用是什么？

报价单用于给客户提供产品、价格、数量、金额等报价信息。

报价单通常来源于：

* 客户需求
* 商机
* 项目
* 销售手工创建

报价单审批通过并被客户确认后，可以转为销售订单。

---

## 2. 报价单状态有哪些？

常见状态：

```text
草稿
待审核
已审核
已驳回
已确认
已取消
```

具体状态以系统实际配置为准。

---

## 3. 报价单可以修改吗？

通常规则：

* 草稿状态可以修改。
* 被驳回后可以修改后重新提交。
* 已提交审批后不建议直接修改。
* 已确认或已转销售订单后不允许随意修改。

如需重新报价，建议复制原报价单后生成新报价。

---

## 4. 报价单如何转销售订单？

操作路径：

```text
报价单详情 → 转销售订单
```

转订单后：

* 客户信息带入销售订单
* 商品明细带入销售订单
* 报价金额带入销售订单
* 报价单与销售订单形成关联

---

## 5. 为什么报价单不能提交？

可能原因：

* 客户为空
* 商品明细为空
* 数量小于等于 0
* 单价为空
* 没有审批权限
* 当前状态不允许提交
* 系统配置未开启相关流程
* 模块授权不包含销售或报价功能

---

# 六、销售订单使用说明

## 1. 销售订单的作用是什么？

销售订单用于确认客户最终购买的商品、数量、金额、交付信息和后续库存、财务流程。

销售订单是连接以下业务的核心单据：

```text
报价
库存检查
缺货采购
销售出库
应收
收款
合同
售后
```

---

## 2. 销售订单基本流程是什么？

```text
创建销售订单
  ↓
提交审批
  ↓
审核通过
  ↓
确认订单
  ↓
库存检查
  ↓
库存充足 → 销售出库
  ↓
库存不足 → 缺货采购
  ↓
出库完成
  ↓
应收 / 收款
  ↓
订单完成
```

---

## 3. 什么是库存检查？

库存检查用于判断销售订单中的 SKU 是否有足够可用库存。

系统会检查：

* 订单数量
* 当前可用库存
* 缺货数量
* 是否需要采购

---

## 4. 什么是缺货采购？

当销售订单库存不足时，可以通过“缺货采购”生成采购订单草稿。

流程：

```text
销售订单库存不足
  ↓
缺货采购
  ↓
生成采购订单草稿
  ↓
采购人员完善供应商和采购价
  ↓
采购审批
  ↓
采购入库
  ↓
销售继续出库
```

---

## 5. 销售订单什么时候扣库存？

销售订单本身不直接扣库存。

正确规则：

```text
销售订单确认后不扣库存
出库单确认后才扣库存
```

这样可以保证库存变化有明确出库单和库存流水。

---

## 6. 销售订单可以取消吗？

通常规则：

* 草稿、待审核、已驳回状态可以取消。
* 已确认但未出库的订单可按权限取消。
* 已出库订单不建议直接取消，应通过退货或反向流程处理。
* 已产生财务核销的订单需要谨慎处理。

---

# 七、采购管理使用说明

## 1. 采购订单的作用是什么？

采购订单用于向供应商采购商品。

采购订单可以来源于：

* 手工创建
* 销售缺货采购
* 库存预警
* 项目需求

---

## 2. 采购订单基本流程是什么？

```text
创建采购订单
  ↓
提交审批
  ↓
审核通过
  ↓
采购确认
  ↓
采购入库
  ↓
库存增加
  ↓
生成应付
  ↓
付款核销
```

---

## 3. 采购订单什么时候增加库存？

采购订单本身不增加库存。

正确规则：

```text
采购订单确认后不加库存
入库单确认后才加库存
```

这样可以保证库存增加有明确入库单和库存流水。

---

## 4. 采购订单可以反审核吗？

如果系统启用反审核，并且单据未产生后续业务，可以反审核。

通常以下情况不允许反审核：

* 已入库
* 已付款
* 已生成应付并核销
* 已关联其他不可撤销业务

---

# 八、库存管理使用说明

## 1. 库存现存量是什么？

库存现存量表示当前系统中各 SKU 在各仓库中的库存情况。

通常包含：

* SKU 编码
* SKU 名称
* 商品名称
* 品牌
* 仓库
* 库存数量
* 可用库存
* 占用库存
* 库存状态

---

## 2. 可用库存和库存数量有什么区别？

库存数量表示账面库存总数。

可用库存表示可以用于销售出库的数量。

示例：

```text
库存数量：10
已被销售订单占用：3
可用库存：7
```

---

## 3. 库存流水是什么？

库存流水记录每一次库存变化。

例如：

* 采购入库
* 销售出库
* 盘点盘盈
* 盘点盘亏
* 销售退货
* 采购退货
* 库存调整
* 售后出库

库存流水用于追溯库存为什么增加或减少。

---

## 4. 能不能直接修改库存？

不建议，也不应允许普通用户直接修改库存。

库存变化应通过单据完成：

```text
入库单
出库单
盘点单
退货单
调整单
调拨单
```

---

## 5. 什么是库存盘点？

库存盘点用于核对系统库存和实际库存是否一致。

流程：

```text
创建盘点单
  ↓
选择仓库和盘点范围
  ↓
开始盘点
  ↓
录入实盘数量
  ↓
系统计算差异
  ↓
提交审核
  ↓
审核通过
  ↓
确认调整库存
  ↓
生成库存流水
```

---

# 九、SN 序列号管理

## 1. 什么是 SN？

SN 是设备序列号，用于唯一标识一台设备。

例如服务器、交换机、存储设备等，通常需要用 SN 追踪：

* 入库来源
* 出库去向
* 客户
* 销售订单
* 质保
* 售后
* 维修记录

---

## 2. 哪些商品需要 SN？

是否需要 SN 由 SKU 配置决定。

适合启用 SN 的商品：

```text
服务器
交换机
存储设备
防火墙
核心硬件设备
高价值配件
```

不一定需要 SN 的商品：

```text
普通线材
普通耗材
低价值配件
```

---

## 3. SN 商品入库有什么要求？

如果 SKU 启用 SN，入库时必须录入对应数量的 SN。

系统会校验：

* SN 是否为空
* SN 是否重复
* SN 数量是否与入库数量一致
* SN 是否已存在
* SN 格式是否符合规则

---

## 4. SN 商品出库有什么要求？

出库时必须选择处于“在库”状态的 SN。

系统会校验：

* SN 是否存在
* SN 是否属于当前 SKU
* SN 是否在库
* SN 是否已被出库
* SN 数量是否与出库数量一致

---

## 5. 如何查看 SN 生命周期？

进入：

```text
库存管理 → SN序列号管理 → SN详情
```

可查看：

* 入库记录
* 出库记录
* 销售订单
* 客户
* 质保信息
* 售后记录
* 状态变化

---

# 十、财务管理使用说明

## 1. 应收是什么？

应收表示客户应该支付给公司的金额。

通常来源于：

* 销售订单
* 销售出库
* 保外售后收费

---

## 2. 应付是什么？

应付表示公司应该支付给供应商的金额。

通常来源于：

* 采购订单
* 采购入库
* 供应商费用

---

## 3. 收款单是什么？

收款单用于记录客户实际付款情况。

收款单可以核销应收记录。

规则：

```text
收款金额不能超过未收金额
已收款应收不能重复收款
取消收款后应反冲金额
```

---

## 4. 付款单是什么？

付款单用于记录公司向供应商实际付款情况。

付款单可以核销应付记录。

规则：

```text
付款金额不能超过未付金额
已付款应付不能重复付款
取消付款后应反冲金额
```

---

## 5. 什么是核销？

核销就是把收款或付款与应收、应付进行匹配。

示例：

```text
客户应收 10000 元
客户付款 6000 元
核销后：
已收 6000
未收 4000
状态：部分收款
```

# 十八、常见问题 QA

## Q1：系统里显示有库存，为什么不能出库？

可能原因：

1. 库存被销售订单占用。
2. 可用库存不足。
3. 库存被冻结。
4. SKU 已停用。
5. SN 商品未绑定 SN。
6. SN 状态不是在库。
7. 当前仓库不匹配。

请查看：

```text
库存现存量
库存流水
SN详情
销售订单库存检查结果
```

---

## Q2：销售订单为什么不能生成出库单？

可能原因：

1. 销售订单未审核。
2. 销售订单未确认。
3. 库存不足。
4. 商品明细为空。
5. 当前用户没有出库权限。
6. 订单已取消或已完成。
7. SN 商品未选择 SN。

---

## Q3：销售订单库存不足怎么办？

可以使用：

```text
销售订单 → 缺货采购
```

系统会根据缺货数量生成采购订单草稿。

采购入库后，再继续销售出库。

---

## Q4：采购入库后库存为什么没有增加？

请检查：

1. 入库单是否已确认。
2. 入库是否被取消。
3. 入库仓库是否正确。
4. 入库 SKU 是否正确。
5. 是否生成库存流水。
6. SN 商品是否完成 SN 建档。

---

## Q5：出库后库存为什么没有减少？

请检查：

1. 出库单是否已确认。
2. 是否选择正确仓库。
3. 是否选择正确 SKU。
4. 是否生成库存流水。
5. 出库单是否被取消。
6. SN 商品是否完成 SN 绑定。

---

## Q6：为什么收款后应收状态没有变？

可能原因：

1. 收款单未确认。
2. 收款未核销应收。
3. 收款金额小于应收金额，状态为部分收款。
4. 收款被取消。
5. 核销记录被反冲。
6. 页面缓存未刷新。

---

## Q7：为什么付款后应付状态没有变？

可能原因：

1. 付款单未确认。
2. 付款未核销应付。
3. 付款金额小于应付金额。
4. 付款被取消。
5. 核销反冲。
6. 应付记录关联错误。

---

## Q8：为什么普通销售看不到某个客户？

可能原因：

1. 客户不是该销售负责人。
2. 客户在数据公海中。
3. 客户被转移给其他销售。
4. 销售没有该客户数据权限。
5. 客户被停用。
6. 当前角色权限配置不正确。

---

## Q9：为什么主管能看到团队客户，而销售只能看到自己的？

这是数据范围控制。

常见规则：

```text
普通销售：本人数据
销售主管：团队数据
管理员/老板：全部数据
```

---

## Q10：为什么我导出的数据比页面少？

可能原因：

1. 导出受当前筛选条件影响。
2. 当前用户没有全部数据权限。
3. 当前用户没有敏感字段权限。
4. 导出数量超过系统限制。
5. 某些字段被导出权限过滤。

---

## Q11：为什么无法删除客户？

可能原因：

1. 客户已有报价单。
2. 客户已有销售订单。
3. 客户已有合同。
4. 客户已有应收或收款。
5. 客户已有售后工单。
6. 当前用户没有删除权限。

建议使用：

```text
停用客户
退回公海
转移客户
```

而不是直接删除。

---

## Q12：为什么无法删除 SKU？

可能原因：

1. SKU 已被报价单引用。
2. SKU 已被销售订单引用。
3. SKU 已被采购订单引用。
4. SKU 已产生库存。
5. SKU 已产生库存流水。
6. SKU 已有关联 SN。

建议使用：

```text
停用 SKU
```

---

## Q13：为什么 SN 无法出库？

可能原因：

1. SN 不存在。
2. SN 已出库。
3. SN 被冻结。
4. SN 状态不是在库。
5. SN 不属于该 SKU。
6. SN 数量与出库数量不匹配。
7. 同一请求中 SN 重复。

---

## Q14：为什么验证码不正确？

可能原因：

1. 验证码已过期。
2. 输入错误。
3. Redis 未启动或连接异常。
4. 浏览器缓存问题。
5. 多次刷新后使用了旧验证码。

建议刷新验证码后重新输入。

---

## Q15：为什么代登录后不能操作？

如果当前是只读代登录模式，系统只允许查看，不允许新增、修改、删除、审批、出入库、收付款等操作。

这是为了避免管理员代登录时误操作业务数据。

---

## Q16：为什么系统提示授权无效？

可能原因：

1. 未上传授权文件。
2. 授权文件已过期。
3. 机器码不匹配。
4. 授权文件被修改。
5. 当前模块未授权。
6. 授权用户数超限。

请进入：

```text
系统管理 → 系统授权
```

查看授权状态。

---

## Q17：为什么无法新增用户？

可能原因：

1. 当前用户没有用户新增权限。
2. 授权最大用户数已达到上限。
3. 用户名重复。
4. 所属机构不存在。
5. 角色未分配。

---

## Q18：为什么看板数据和列表数据有一点延迟？

看板和报表可能使用缓存。

一般缓存时间为几十秒到几分钟。

如果刚操作完业务，列表通常是最新数据，看板可能稍后刷新。

---

## Q19：为什么导入客户时失败？

常见原因：

1. 客户名称为空。
2. 客户名称重复。
3. 负责人账号不存在。
4. 导入目标为空。
5. 导入到我的客户但未填写负责人。
6. 导入到数据公海但填写了负责人。
7. 标签格式错误。
8. 手机号格式错误。

---

## Q20：为什么客户退回公海后，我的客户里看不到了？

客户退回公海后，客户归属状态变为公海客户，不再属于当前销售。

可以在：

```text
CRM中心 → 数据公海
```

中查看。

# 环境变量覆盖配置（ENV > YAML）

系统支持通过环境变量覆盖配置文件中的参数。

优先级：

```text
环境变量（ENV）
    ↓
config.yaml
    ↓
系统默认值
```

例如：

```yaml
mysql:
  host: 127.0.0.1
  port: 3306
```

可通过环境变量覆盖：

```bash
export MYSQL_HOST=10.0.0.10
export MYSQL_PORT=3306
```

启动后实际生效：

```yaml
mysql:
  host: 10.0.0.10
  port: 3306
```

推荐支持的环境变量：

| 环境变量           | 对应配置           |
| -------------- | -------------- |
| APP_ENV        | app.env        |
| APP_PORT       | app.port       |
| MYSQL_HOST     | mysql.host     |
| MYSQL_PORT     | mysql.port     |
| MYSQL_DATABASE | mysql.database |
| MYSQL_USERNAME | mysql.username |
| MYSQL_PASSWORD | mysql.password |
| REDIS_HOST     | redis.host     |
| REDIS_PORT     | redis.port     |
| REDIS_PASSWORD | redis.password |
| JWT_SECRET     | jwt.secret     |
| LOG_LEVEL      | log.level      |

Docker 场景：

```bash
docker run \
-e MYSQL_HOST=mysql \
-e MYSQL_PASSWORD=123456 \
-e JWT_SECRET=xxxxxxxx \
hardware-erp-api
```

## 数据表设计

无需：

```sql
tenant_id
merchant_id
shop_id
```

例如：

```sql
CREATE TABLE products (
    id BIGINT PRIMARY KEY,
    sku VARCHAR(50),
    name VARCHAR(200),
    stock_quantity DECIMAL(18,2)
);
```

而不是：

```sql
CREATE TABLE products (
    id BIGINT PRIMARY KEY,
    tenant_id BIGINT,
    sku VARCHAR(50),
    name VARCHAR(200)
);
```

# Nginx 反向代理配置

配置文件：

```text
/etc/nginx/conf.d/hardware-erp.conf
```

示例：

```nginx
server {
    listen 80;
    server_name erp.example.com;

    client_max_body_size 100m;

    location / {
        proxy_pass http://127.0.0.1:8080;

        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;

        proxy_set_header X-Forwarded-For
        $proxy_add_x_forwarded_for;

        proxy_set_header X-Forwarded-Proto
        $scheme;
    }
}
```

---

## HTTPS 配置

```nginx
server {
    listen 443 ssl http2;
    server_name erp.example.com;

    ssl_certificate
    /etc/letsencrypt/live/erp/fullchain.pem;

    ssl_certificate_key
    /etc/letsencrypt/live/erp/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8080;
    }
}
```

测试：

```bash
nginx -t
```

重载：

```bash
systemctl reload nginx
```

# Systemd 服务部署（生产环境）

创建服务文件：

```bash
/etc/systemd/system/hardware-erp.service
```

内容：

```ini
[Unit]
Description=Hardware ERP API
After=network.target mysql.service redis.service

[Service]
Type=simple

User=erp
Group=erp

WorkingDirectory=/opt/hardware-erp

ExecStart=/opt/hardware-erp/hardware-erp-api \
--config=/opt/hardware-erp/configs/config.yaml

Restart=always
RestartSec=5

LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
```
